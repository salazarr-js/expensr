All record types should be reachable via the quick record input and iPhone Shortcut — not just expenses.

**Why:** The user values frictionless recording. If a feature requires opening a specific form/modal, it's not fast enough. Quick record should be the universal entry point.

**How to apply:** When adding new record types or features, always consider how they'd work through the smart parse input. Plan the natural language patterns (e.g. "transfer 3500 galicia → cash", "pago angy 5000") alongside the form UI.
